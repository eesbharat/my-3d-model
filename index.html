Tip: upload index.html and your model.glb to the same repository root. Then enable GitHub Pages on that repo. The page will be available at https://<your-username>.github.io/<repo-name>/

Step-by-step (non-coder friendly)
  1. Create a new repository on GitHub
    • Go to https://github.com and sign in.
    • Click the green New button (top-left under your profile or on your profile page).
    • Give the repo a simple name, e.g. my-3d-model. Description optional.
    • Set it to Public (easiest for Pages). Click Create repository.
  2. Upload files (index.html + your .glb)
    • Open your repo on GitHub. Click Add file → Upload files.
    • Drag & drop your index.html (this file) and your model.glb file into the browser area.
    • Scroll down and click Commit changes.
  3. Enable GitHub Pages
    • In your repo, click Settings → left menu click Pages (or find Pages under Code and automation).
    • Under Build and deployment, choose Branch: main and / (root) folder.
    • Click Save or Save and Deploy. GitHub will show the URL (it may take a minute to deploy).
  4. Open your page
    • Visit: https://<your-username>.github.io/<repo-name>/ (replace placeholders with your username and repo name).
    • If the model doesn’t appear, check the browser console (right-click → Inspect → Console) for a 404 filename mismatch or CORS errors.
  5. Troubleshooting
    • Filename: make sure the HTML’s src="model.glb" matches the exact filename (case-sensitive).
    • Large files: GitHub has a 100 MB file limit in the web UI. If your .glb is bigger, use Git LFS or reduce size in Blender.
    • Privacy: Public repos are easiest. Private repos require additional steps (or a paid plan for Pages on private repos).
